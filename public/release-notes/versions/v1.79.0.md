### Upgrade impact
- Existing configurations are likely unaffected unless you use outbound contact lists or templates with timezone columns, task management worktypes with `disable_default_status_creation`, non-internal DNC lists, knowledge document variations, or provider export.
- Three new resources are available: `genesyscloud_speechandtextanalytics_topic`, `genesyscloud_customer_intent`, and `genesyscloud_intent_category`.
- Outbound contact list and template resources gain `*_time_column_name` attributes; legacy `*_time_column` attributes remain but are deprecated. State is migrated automatically on upgrade. Prefer the new attribute names in new or updated configuration.
- Plans for contact lists and templates should show fewer spurious diffs for timezone columns, especially when `automatic_time_zone_mapping` is enabled.
- `genesyscloud_task_management_worktype` should no longer show perpetual diffs on `disable_default_status_creation`.
- Deleting or updating `genesyscloud_outbound_dnclist` entries behaves differently for non-`rds` lists; entry uploads are blocked and phone-entry cleanup on delete is skipped.
- Deleting `genesyscloud_knowledge_document_variation` should succeed when sibling variations remain.
- `genesyscloud_routing_email_domain` data source lookups accept case-insensitive names and subdomain prefixes.
- Users running `genesyscloud_tf_export` may see different HCL for evaluation forms, surveys, routing email domains, and outbound contact lists; export is more resilient to transient API errors.
