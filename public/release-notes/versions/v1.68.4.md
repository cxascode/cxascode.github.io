#### genesyscloud_guide

Removed:
- `source` — required attribute that selected how guide content was generated (`Manual`, `Prompt`, or `Document`).
- `prompt` — optional prompt used when `source` was `Prompt`.
- `url` — optional URL used when `source` was `Prompt`.
- `status` — computed guide status (`Draft` or `ProductionReady`).
- `latest_saved_version` — computed latest saved version identifier.
- `latest_production_ready_version` — computed latest production-ready version identifier.

- Create behavior — create no longer runs prompt-based content generation or automatically creates an initial guide version; manage versions with `genesyscloud_guide_version` separately.

#### genesyscloud_guide_version

Added:
- None detected.

Removed:
- `state` — optional attribute that controlled draft vs publish workflow (`Draft`, `ProductionReady`, or `TestReady`).

Changed:
- Create behavior — after create, the provider always attempts to publish the version to ProductionReady; draft-only creation via `state = "Draft"` is no longer supported.
- Update behavior — when the parent guide already has a production-ready version, update creates a new version instead of patching the existing one; otherwise it updates the latest saved version.
- Update behavior — after update, the provider always attempts to publish the version to ProductionReady.
- Export behavior — export includes only each guide's latest saved version (no longer exports a separate production-ready version when it differs).
- Export behavior — export resolves `resources.data_action.data_action_id` references to `genesyscloud_integration_action` resources.
- Export behavior — export omits `generate_content` from generated configuration.

### Upgrade impact
- Remove `source`, `prompt`, `url`, `status`, `latest_saved_version`, and `latest_production_ready_version` from `genesyscloud_guide` configurations; plans will fail if these arguments remain.
- Remove `state` from `genesyscloud_guide_version` configurations; create and update now always publish to ProductionReady.
- Existing `genesyscloud_guide` resources that relied on Prompt-based generation or automatic initial version creation need to be reworked: create the guide with only `name`, then manage content with `genesyscloud_guide_version`.
- `genesyscloud_guide_version` updates on guides that already have a production-ready version may create a new version and change the resource ID; review plans carefully before apply.
- Export output for guide versions may differ: fewer versions exported per guide, integration action references resolved, and `generate_content` omitted.
