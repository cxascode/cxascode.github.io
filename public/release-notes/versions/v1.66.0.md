### Upgrade impact
- `genesyscloud_guide` is newly available for managing Genesys Cloud guides.
- Phone number validation across resources that use E.164 checks now requires a leading `+` and accepts any viable E.164 number rather than requiring an exact canonical format match; validation is skipped when BCP mode is enabled. Review configurations that relied on strict format normalization at plan time.
- `genesyscloud_user` — if you omit `employer_info` or remove `addresses` from configuration, plan/apply behavior changes: employer info will no longer appear in state from API reads, and removed address blocks will clear addresses in Genesys Cloud.
- Bulk export block labels change for architect flows, knowledge resources, and users; re-exported configurations may show different resource block names.
- Bulk export for scripts no longer includes Genesys Cloud default scripts.
- `genesyscloud_routing_email_route` read and `genesyscloud_routing_email_domain` / `genesyscloud_script` delete behavior is more tolerant of already-absent remote objects.
- Contact list bulk export may complete sooner due to revised export URL polling.
- Most other existing configurations are likely unaffected.
