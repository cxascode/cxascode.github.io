### Resources changed

#### genesyscloud_outbound_contact_list

Added:
- `whats_app_columns` — optional set block (forces replacement when changed) to designate which contact-list columns hold WhatsApp contacts. At least one of `phone_columns`, `email_columns`, or `whats_app_columns` must be configured.
- `whats_app_columns.column_name` — required column name for a WhatsApp contact field.
- `whats_app_columns.type` — required WhatsApp column type (for example, `work` or `personal`).

Removed:
- None detected.

Changed:
- `phone_columns` — description updated; this block is required when both `email_columns` and `whats_app_columns` are empty.
- `email_columns` — description updated; this block is required when both `phone_columns` and `whats_app_columns` are empty.
- Export behavior — exported configurations include `whats_app_columns` blocks when WhatsApp columns exist on the contact list.

### Upgrade impact
- Most existing configurations are likely unaffected.
- `genesyscloud_outbound_contact_list` gains optional `whats_app_columns` support; existing contact lists without WhatsApp columns require no configuration changes.
- If you explicitly set `max_concurrent_threads` on `genesyscloud_tf_export`, that setting now takes effect instead of being overridden by provider `max_clients`.
- Architect flow export workflows may be more reliable under API rate limiting and when exporting published flow versions.
