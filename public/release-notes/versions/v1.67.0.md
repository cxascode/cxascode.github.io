### Upgrade impact
- `genesyscloud_integration_webhook` is newly available for reading webhook integration IDs and invocation URLs by integration name.
- `genesyscloud_auth_role` create/update is more permissive when permission policies cannot be validated; review roles that previously failed at plan/apply time because of missing product entitlements.
- `genesyscloud_task_management_worktype` updates to `default_expiration_seconds` now take effect correctly; re-apply if you relied on that attribute.
- Task management worktype, status, and status transition resources retry HTTP 409 responses, which can reduce transient apply failures under concurrent API use.
- Bulk export output changes for integrations (flow references), locations, utilization labels, phones, trunk base settings, and overall export concurrency/error reporting; re-exported configurations may differ.
- Shared retry helpers retry a broader set of timeout conditions across resources that use them.
- Most other existing configurations are likely unaffected.
