### Resources changed

#### genesyscloud_outbound_messagingcampaign

Added:
- `whats_app_config` — Optional block to configure WhatsApp messaging for the campaign; exactly one of `email_config`, `sms_config`, or `whats_app_config` must be set.
- `whats_app_config.whats_app_columns` — Contact list columns that specify the WhatsApp address(es) for each contact.
- `whats_app_config.whats_app_integration_id` — WhatsApp integration used to send messages to contacts.
- `whats_app_config.content_template_id` — Content template used to formulate the WhatsApp message sent to contacts.

Removed:
- None detected.

Changed:
- `email_config` — Exactly-one validation now requires exactly one of `email_config`, `sms_config`, or `whats_app_config`.
- `sms_config` — Exactly-one validation now requires exactly one of `email_config`, `sms_config`, or `whats_app_config`.
- `whats_app_config` — Populated from the Genesys Cloud API during read, plan, and apply refresh.
- Export behavior — `whats_app_config.content_template_id` and `whats_app_config.whats_app_integration_id` are resolved as Terraform references during export.

### Upgrade impact
- Existing Terraform configurations are likely unaffected unless you manage outbound messaging campaigns or use provider export.
- `genesyscloud_outbound_messagingcampaign` now supports WhatsApp configuration alongside email and SMS; campaigns using only email or SMS remain valid, but you can now choose WhatsApp as the single messaging channel.
- A new `genesyscloud_knowledge_document` data source lets you reference documents by title and knowledge base name instead of managing IDs directly.
- Users running `genesyscloud_tf_export` may see different exported HCL for knowledge resources, response management responses, outbound messaging campaigns with WhatsApp config, and integration credentials (credential `name` values export as raw prefixed strings again rather than integration references).
