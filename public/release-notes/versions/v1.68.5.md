### Resources changed

#### genesyscloud_flow

Added:
- `update_trigger_hash` — optional hash used to trigger resource updates when external values change; use as an alternative to `file_content_hash` for finer control over when updates run.

Removed:

#### genesyscloud_oauth_client

Removed:

Changed:
- `client_secret` — populated in Terraform state only when `expose_client_secret` is `true`; otherwise remains empty in state.
- Read behavior — `client_id` is now populated during read operations.

### Provider configuration changes
- Added support for the `mx-central-1` region (maps to `mxc1.pure.cloud`).
- Added support for the `ap-southeast-1` region (maps to `apse1.pure.cloud`).

### Upgrade impact
- Remove `file_content_hash` from `genesyscloud_flow` configurations; use `update_trigger_hash` instead to control when flow updates are triggered. Plans will fail if `file_content_hash` remains set as an input argument.
- Existing `genesyscloud_flow` resources that relied on setting `file_content_hash` should migrate to `update_trigger_hash` with an equivalent hash expression to preserve update behavior.
- `genesyscloud_oauth_client` configurations are unaffected unless you opt in by setting `expose_client_secret = true`, which stores the client secret in state.
- `genesyscloud_outbound_contact_list` resources with an empty `contacts_filepath` will no longer error during bulk contact upload; the upload step is skipped instead.
- Export output for `genesyscloud_knowledge_document` may differ: `label_names` now reference `genesyscloud_knowledge_label` resources.
- Provider configurations using `mx-central-1` or `ap-southeast-1` as the region value are now supported.
