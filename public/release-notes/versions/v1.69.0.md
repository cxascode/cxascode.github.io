### Upgrade impact
- Existing configurations are likely unaffected for most resources; review file-based resources if you manage content via S3 paths or `file_content_hash`.
- `genesyscloud_routing_queue` can now manage Conditional Group Activation inline via `conditional_group_activation`.
- `genesyscloud_script` and `genesyscloud_responsemanagement_responseasset` compute `file_content_hash` automatically; remove it from configuration if you set it manually, and expect a warning if you continue to set it.
- `genesyscloud_architect_grammar_language` still requires `file_content_hash` for non-S3 file paths even though the attribute is optional in the schema.
- `genesyscloud_outbound_contact_list` applies may take longer when uploading contacts because the provider waits for import completion; uploads are skipped when `contacts_filepath` is empty.
- `genesyscloud_organization_authentication_settings` plans may show new defaults for `password_requirements` and reject values below the documented minimums.
- `genesyscloud_outbound_digitalruleset` configurations with `Numeric` or `Period` condition types must include a `value`; `String` and `DateTime` conditions may omit it.
- `genesyscloud_outbound_ruleset` dialer actions can use `APPEND_CUSTOM_ENTRY_TO_DNC_LIST`.
- Export output may differ for external contacts resources, flows, scripts, response assets, digital rulesets, routing queues with conditional group activation, and users with zero routing utilization capacity.
