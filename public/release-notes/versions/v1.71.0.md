### Upgrade impact
- Existing configurations are likely unaffected for most resources.
- `genesyscloud_business_rules_decision_table` supports `stringList` literals and additional expression comparators; decision tables using list-based conditions can use the new `defaults_to.values` and `stringList` literal type.
- `genesyscloud_business_rules_schema` resources created with `enabled = false` should now receive the full configured state after create.
- `genesyscloud_responsemanagement_responseasset` can set an optional `name` separate from `filename`; plans may show `filename` no longer being replaced on read by the API asset name.
- `genesyscloud_routing_queue` callback settings can configure customer-first callback retries via `media_settings_callback.max_retry_count` and `media_settings_callback.retry_delay_seconds`. Removing `bullseye_rings` from a queue configuration triggers a pre-update cleanup of bullseye `member_groups`.
- `genesyscloud_script` create operations no longer fail early on duplicate names; rely on API errors or unique naming in your configuration.
- `genesyscloud_telephony_providers_edges_site` updates are less likely to clear API fields that are not defined in Terraform.
- `genesyscloud_tf_export` can tune export parallelism with `max_concurrent_threads`; adjust if exports are too slow or consume too many API tokens.
