### Resources changed

#### genesyscloud_integration_action

Added:
- `function_config` — optional block for function-based integration actions (categories whose name contains "function data action").
- `function_config.description` — description of the function; optional and computed.
- `function_config.handler` — handler function name; optional and computed.
- `function_config.runtime` — runtime environment for the function; optional and computed.
- `function_config.timeout_seconds` — function execution timeout in seconds; optional and computed.
- `function_config.zip_id` — ID of the uploaded zip file containing function code; optional and computed.
- `function_config.file_path` — local path to the zip file containing function code; required when using `function_config` for create or update.

Removed:
- None detected.

Changed:
- `config_request.request_url_template` — no longer required; now optional and computed.
- Create behavior — integration actions in a function data action category are created through a draft-and-publish workflow that uploads the zip from `function_config.file_path`, applies function settings, and publishes the draft; `function_config.file_path` is required for these actions.
- Update behavior — function data action categories follow the same draft workflow to upload an updated zip and publish changes.
- Read behavior — refreshes `function_config` from the API when the action category is a function data action.
- Read behavior — consistency state validation is no longer performed on refresh, which may surface plan diffs that were previously suppressed.

### Upgrade impact
- `genesyscloud_integration_action` now supports function data actions via the `function_config` block; set `function_config.file_path` to a zip containing your function code when creating or updating actions in that category.
- Existing non–function-data-action integration actions are likely unaffected, though `config_request.request_url_template` is no longer required in configuration and may be omitted when the API supplies a value.
- `genesyscloud_quality_forms_survey` name lookups should be more reliable after the lookup fix; no configuration changes are required.
- Set `ROUTING_QUEUE_DELAY_MAX` only if you need to throttle routing queue API calls; otherwise existing `genesyscloud_routing_queue` configurations are likely unaffected aside from normal provider timing.
