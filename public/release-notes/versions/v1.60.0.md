### Upgrade impact
- `genesyscloud_group` gains the optional `calls_enabled` argument (default `true`). Existing group configurations should continue to work without changes.
- `genesyscloud_routing_queue` resources that use `media_settings_message` may show plan diffs after upgrade because state is mapped more accurately to the API.
- `genesyscloud_routing_email_route` refresh behavior is more robust when the API returns empty or missing route lists; resources are no longer silently dropped from state when a route is temporarily unavailable.
- Exporting `genesyscloud_integration_credential` resources now generates block labels in the form `Integration-{integration_name}` instead of `{integration_name}_{credential_name}`.
- Exporting `genesyscloud_integration_action` resources now preserves zero-value entries in `config_response.translation_map_defaults`.
- Bulk resource export limits concurrent API operations to the configured `token_pool_size`, which can affect large export runs.
