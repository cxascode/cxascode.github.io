### Upgrade impact
- Most existing configurations continue to work without changes.
- Review `genesyscloud_architect_ivr` resources that rely on API-populated `dnis` values; you may need to add explicit `dnis` blocks or accept plan diffs after upgrade.
- `genesyscloud_journey_segment` gains optional `assignment_expiration_days` and deprecates `scope`; no immediate action is required, but plan to remove `scope` before a future provider release drops it.
- `genesyscloud_routing_queue` callback and canned-response settings may show plan diffs; new callback attributes are optional and `media_settings_callback.mode` is now populated from the API when not configured.
- `genesyscloud_task_management_worktype` can now manage `default_script_id`; existing worktypes without this attribute are unaffected until you choose to set it.
- Destroying a `genesyscloud_routing_email_route` that the API already removed (404) now completes successfully instead of failing.
- Destroying `genesyscloud_telephony_providers_edges_trunkbasesettings` retries transient HTTP 400 responses for up to 50 seconds before failing.
- Bulk export for `genesyscloud_routing_queue` resolves new callback flow ID references and lists queues more reliably; bulk export for `genesyscloud_telephony_providers_edges_site` no longer resolves `outbound_routes.external_trunk_base_ids` to trunk base settings references.
