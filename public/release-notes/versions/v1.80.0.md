### Upgrade impact
- **Breaking for intent resources:** Rename `genesyscloud_customer_intent` and `genesyscloud_intent_category` (and their data sources) to `genesyscloud_intents_customerintents` and `genesyscloud_intents_categories` before upgrading if you manage customer intents with Terraform.
- **Journey Outcomes deprecation:** `genesyscloud_journey_outcome`, `genesyscloud_journey_outcome_predictor`, and related journey action map outcome trigger attributes are deprecated; plan to remove them from configuration.
- **Routing queue standalone modes:** Set `ENABLE_STANDALONE_CGR` to use `genesyscloud_routing_queue_conditional_group_routing` instead of `conditional_group_routing_rules` on the queue. Use `genesyscloud_routing_queue_conditional_group_activation` for conditional group activation when standalone CGA mode is enabled.
- **Outbound campaigns:** `max_calls_per_agent` now accepts floats; existing integer values remain valid. New `diagnostics_settings` is only available for power and predictive dialing modes.
- **Conditional group activation groups:** `conditional_group_activation.rules.groups` changed from list to set; Terraform may reorder blocks without functional change.
- **SDK debug users:** Enabling `sdk_debug` produces more verbose hook output and more complete error logs in `sdk_debug_file_path`.
- Configurations not using the changed resources are likely unaffected.
