### Upgrade impact
- Existing `genesyscloud_routing_queue` configurations without `bullseye_rings` are unaffected.
- Queues that use `bullseye_rings` may show plan or state differences: the provider no longer injects a hidden default sixth ring on apply, and the last ring returned by the API is now kept in Terraform state.
- Configurations with only one `bullseye_rings` block will fail validation; add at least two blocks or remove the attribute.
- If you previously relied on the provider to add the default final ring, define that ring explicitly in configuration.
