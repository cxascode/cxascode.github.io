### Upgrade impact
- `genesyscloud_integration_action_draft` is newly available for managing draft integration actions.
- `genesyscloud_group` adds optional `include_owners` (defaults to `true`); existing configurations are likely unaffected.
- `genesyscloud_outbound_messagingcampaign` now requires exactly one of `email_config` or `sms_config`; email campaigns are fully supported. Review campaigns that previously omitted both blocks or relied on email settings that were ignored.
- `genesyscloud_quality_forms_evaluation` changes how publishing affects Terraform state: after publish, the resource ID becomes the new draft ID and `published_id` holds the published version. Refresh `published` expectations if you relied on it being synced on every read.
- `genesyscloud_organization_presence_definition` accepts additional `system_presence` values and no longer validates `language_labels` map keys at plan time.
- `genesyscloud_webdeployments_configuration` adds push-notification label keys and stricter messenger color validation.
- Bulk export block labels may change for IVRs and response management responses.
- Bulk export now supports `genesyscloud_flow_loglevel`; routing queue exports no longer resolve SMS address IDs.
- Most other existing configurations are likely unaffected.
