### Resource behavior

### Export behavior

- Deprecated schema attributes are omitted from exported configuration when `export_deprecated` is set to `false`; the default `true` preserves prior export output that included deprecated attributes.
- Schema-based attribute exclusions for computed, read-only computed, and deprecated fields now apply recursively to nested block attributes, and are applied before output sanitization.

### Related export output changes

- Exported configuration for any resource type may omit nested block attributes that are computed, read-only computed, or deprecated, where those nested fields were previously left in place.
- When `export_deprecated` is set to `false`, deprecated attributes are omitted from exported configuration across all exported resource types that define them.

### Upgrade impact

- Existing `genesyscloud_tf_export` configurations are likely unaffected; `export_deprecated` defaults to `true`, matching prior behavior.
- Re-running exports after upgrade may produce slightly different HCL for resources with nested computed, read-only, or deprecated block attributes due to recursive exclusion.
- Set `export_deprecated = false` if you want exported files to drop deprecated provider attributes; note that changing this setting forces a new export run.
