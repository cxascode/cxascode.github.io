### Export behavior

- `enable_dependency_resolution` — When using `include_filter_resources` or `include_filter_resources_by_id`, setting this to `true` now generates `depends_on` blocks without requiring a non-empty resource-type filter list.
- `enable_dependency_resolution` — When using `exclude_filter_resources` or the default export mode (no include/exclude filter), this setting no longer adds `depends_on` blocks, even when set to `true`.
- Failures while writing custom export files for a resource are now reported as errors and fail the export run instead of being logged as warnings.
- Permission errors encountered while listing resources during export are collected and reported in the end-of-run error summary.

### Related export output changes

- Exported string attributes that embed a resource identifier within a larger value may now keep the surrounding text while substituting only the identifier portion with a Terraform reference.

### Upgrade impact

Existing `genesyscloud_tf_export` configurations are likely unaffected; no resource schema changes were detected. Review export behavior if you rely on `enable_dependency_resolution` with exclude or default export modes, if you depend on custom file writers succeeding with only warnings, or if you export string attributes that embed resource IDs within larger values—the exported HCL may differ from prior releases.
