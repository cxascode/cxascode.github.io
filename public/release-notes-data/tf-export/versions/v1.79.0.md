### Resource behavior

- `exclude_attributes` — Exclusions are applied when exporters are loaded, in addition to the final output pass.
- `exclude_attributes` — Each entry must use `resource_type.attribute` format; malformed entries fail the export with an error.
- `exclude_attributes` — When a listed resource type is not included in the export and `enable_dependency_resolution` is false, the export fails with an error instead of continuing.

### Export behavior

- Resource discovery retries transient API failures up to three times with exponential backoff; permission errors are not retried.
- Resource type retrieval during export is now concurrency-limited (default 10), reducing the likelihood of API rate-limit errors.
- Resource state refresh during export runs concurrently, improving export performance for large environments.

### Related export output changes

- Unresolved GUID values in list-type reference attributes can be mapped to `${data.<type>.<label>.id}` references when a resource exporter defines a custom data-source resolver for that attribute.

### Upgrade impact

Existing `genesyscloud_tf_export` configurations are likely unaffected; no resource schema attributes were added or removed. Re-run exports if you use `exclude_attributes`, because exclusions now apply earlier and invalid or non-exported resource types may fail the run where they previously continued. Export runs should be more resilient to transient API errors and may complete faster. Re-exported HCL may differ for resources whose list attributes contain GUID references that now resolve to data sources.
