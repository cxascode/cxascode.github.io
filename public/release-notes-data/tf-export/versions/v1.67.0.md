### Export behavior

- Bulk export limits concurrent API reads to 10 by default, or to the provider `max_clients` setting when that value is configured.
- Individual resource read failures no longer stop export for the rest of that resource type; successfully read resources are still included in the generated output.
- When one or more resources fail to export, the exporter writes `export_errors.json` in the export directory with per-resource details (type, label, ID, error message, and whether the failure was a timeout).
- After export completes, Terraform warnings summarize timeout and non-timeout failure counts and reference `export_errors.json` for details. Timeout warnings suggest increasing provider `token_pool_size` and/or `token_acquire_timeout`.
- Timeout failures during per-resource reads are retried up to three times with exponential backoff instead of retrying indefinitely; on the first timeout, the exporter attempts to expand the API client pool before retrying.
- Export progress and error messages are emitted through Terraform structured logging when `TF_LOG` is enabled, rather than standard Go log output.

### Upgrade impact

Existing `genesyscloud_tf_export` configurations are unchanged and should not require updates. Exports are more likely to complete with partial output when individual resources fail, so review `export_errors.json` and Terraform warnings after upgrading instead of expecting a single hard failure. If timeout warnings appear, tune provider `max_clients`, `token_pool_size`, and/or `token_acquire_timeout` before re-running the export.
