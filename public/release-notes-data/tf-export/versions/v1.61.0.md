## CX as Code v1.61.0 — genesyscloud_tf_export

[View release changelog](https://github.com/MyPureCloud/terraform-provider-genesyscloud/releases/tag/v1.61.0)

### Resource attributes added

- `use_legacy_architect_flow_exporter` — Optional boolean (defaults to `true`, `ForceNew`). When set to `false`, architect flow configuration files are downloaded as part of the flow export process.

### Resource attributes removed

None detected.

### Resource attributes changed

None detected.

### Resource behavior

- A successful export can now return warning diagnostics to Terraform instead of discarding them; only errors still stop the export.

### Export behavior

- Post-export Terraform state verification results are now returned to the user instead of being ignored.
- Failures from per-resource custom file writers now surface as Terraform warning diagnostics instead of being logged only.
- When `use_legacy_architect_flow_exporter` is `false`, architect flows are processed with the new flow exporter, which downloads flow configuration files during export.
- Exported data source blocks no longer rewrite unresolvable attributes as `${var.*}` placeholder references.

### Related export output changes

- `genesyscloud_architect_flow` — When `use_legacy_architect_flow_exporter` is `false` on `genesyscloud_tf_export`, exported architect flows include downloaded flow configuration files.

### Upgrade impact

- Existing `genesyscloud_tf_export` configurations are likely unaffected because `use_legacy_architect_flow_exporter` defaults to `true`, preserving prior architect flow export behavior.
- Set `use_legacy_architect_flow_exporter` to `false` to enable downloaded architect flow configuration files during export; changing this attribute forces a new export run.
- Export jobs may now show warnings that were previously silent, including state verification issues and custom file writer failures.
