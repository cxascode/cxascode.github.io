### Export behavior

- JSON multi-file export only writes `variables.tf.json` when the export defines variables; an empty variables file is no longer created.
- JSON multi-file export skips resource-type and data-source files when a type has no exported instances.
- Single-file JSON export omits top-level `resource`, `data`, and `variable` sections when those sections would be empty.
- Data source instances are no longer tracked in managed-resource type maps during export assembly, preventing empty managed-resource entries for data-source-only types.
- Resource state reads during export are synchronized, which may reduce intermittent failures when exporting large configurations.

### Related export output changes

- Exported data source blocks no longer include `depends_on` attributes; dependency references are added only to managed resources.

### Upgrade impact

Existing `genesyscloud_tf_export` configurations are unaffected. Re-exported JSON output may omit previously empty files and empty top-level sections. Exported data source configurations may no longer include `depends_on` attributes that appeared in v1.64.1 output.
