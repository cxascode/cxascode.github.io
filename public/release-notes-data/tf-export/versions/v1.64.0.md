### Export behavior

- When tf-export resolves `depends_on` references, dependencies on data sources now include the required `data.` prefix (for example, `data.genesyscloud_<type>.<label>` instead of `genesyscloud_<type>.<label>`).
- When exporting more than one output format in a single run (for example, HCL and JSON together), diagnostics from each export step are now collected and returned together instead of only the last step's result being reported.

### Related export output changes

- Exported `depends_on` values that reference data sources may differ from v1.63.0 output because the `data.` prefix is now included where it was previously omitted.

### Upgrade impact

Existing `genesyscloud_tf_export` configurations are unaffected. Re-exported configuration that includes `depends_on` references to data sources may show corrected `data.`-prefixed references. Multi-format export runs may surface additional or combined error messages when more than one export step fails.
