### Upgrade impact
- Two new resources and matching data sources are available: `genesyscloud_speechandtextanalytics_dictionaryfeedback` and `genesyscloud_workforcemanagement_businessunits`.
- `genesyscloud_quality_forms_evaluation` supports additional question types and multiple-select question structures; existing forms that relied on always-required `answer_options` may need configuration updates.
- `genesyscloud_webdeployments_configuration` can configure journey tracking filters and URL/query behavior under `journey_events.tracking_settings`.
- `genesyscloud_telephony_providers_edges_extension_pool` accepts an optional `division_id`.
- `genesyscloud_oauth_client` no longer stores `client_secret` in state; do not rely on Terraform state to retrieve secrets after upgrade.
- `genesyscloud_auth_division` destroy no longer fails when the division was already deleted manually.
- `genesyscloud_script` drops removed scripts from state on refresh instead of leaving a stale entry.
- `genesyscloud_team` may show updated `member_ids` after partial member-add failures and requires a follow-up apply to retry failed members.
- `genesyscloud_location` updates are less likely to fail when a location has an emergency number and you change non-address fields.
- Flow file download during architect flow operations is more resilient to API rate limits; no schema changes are required.
