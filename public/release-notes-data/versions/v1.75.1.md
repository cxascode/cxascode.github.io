### Upgrade impact
- This release primarily affects `genesyscloud_knowledge_document_variation`. Existing configurations without tables, nested lists, or multiple context values are likely unaffected.
- Table blocks, nested lists, video content in list items, and `priority` are now supported; review variations that use complex document bodies before upgrading.
- Variations with more than one context entry or more than one value per context will now persist all entries instead of only the first.
- Deeply nested lists (beyond 9 levels) or nested tables (beyond 1 level, or 3 levels with `ENABLE_NESTED_TABLES` set) will fail on create, update, or refresh.
- Paragraph and list blocks must include their `blocks` child blocks; configurations that omitted them may require updates.
- Re-exporting knowledge document variations omits `document_version` from generated configuration.
