### Upgrade impact
- Three resources are newly available: `genesyscloud_organization_presence_definition`, `genesyscloud_quality_forms_evaluation`, and `genesyscloud_quality_forms_survey`.
- `genesyscloud_flow_outcome` create now adopts existing outcomes by name; importing or creating duplicate names should no longer fail outright.
- `genesyscloud_journey_segment` no longer accepts `scope`; remove it from configuration.
- `genesyscloud_externalcontacts_organization` no longer supports `primary_contact_id`; remove it from configuration.
- `genesyscloud_outbound_campaign` no longer requires `caller_name` or `caller_address`.
- `genesyscloud_routing_queue` adds `ignore_members` for stacks that manage queue members outside Terraform; review `members` syntax if you previously assigned members as a bare attribute.
- `genesyscloud_routing_email_route` has stricter validation and changed reply-address handling; review configurations that mix `from_email`, `auto_bcc`, and `reply_email_address`.
- `genesyscloud_user` may show a new computed `addresses.phone_numbers.extension_pool_id` after upgrade; bulk export resolves it to `genesyscloud_telephony_providers_edges_extension_pool` references.
- Bulk export `depends_on` references to data sources now use the `data.` prefix.
- Bulk export for `genesyscloud_routing_sms_address` may use `postal_code` as the block label when `name` is absent.
- Most other existing configurations are likely unaffected.
