### Provider configuration changes
- `custom_retry_timeout` — optional duration (default `5m`) for how long the provider retries read operations when a resource in state returns 404 from the API before removing it from state. Set to `0` or `0s` for immediate fail-fast behavior. Can also be set with the `GENESYSCLOUD_CUSTOM_RETRY_TIMEOUT` environment variable.

### Upgrade impact
- Most existing configurations are likely unaffected; the release focuses on read/update reliability, export output, and optional provider tuning.
- Set `custom_retry_timeout = "0s"` if you need resources deleted outside Terraform to drop out of state immediately on the next refresh instead of being retried for up to five minutes.
- `genesyscloud_user` configurations with extension-only phone numbers may see fewer perpetual diffs; `phone_numbers.number` and `phone_numbers.extension_pool_id` may appear as computed values in state after upgrade.
- `genesyscloud_outbound_dnclist` refresh may normalize `entries` ordering and grouping; non-internal lists will show empty `entries` in state.
- `genesyscloud_outbound_digitalruleset` configurations that omit `rules` are now valid.
- Export workflows for flows, routing queue member groups, and in-queue email flow dependencies may produce more consistent references and sorted dependency metadata.
