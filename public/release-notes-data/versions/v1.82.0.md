### Upgrade impact
- Five new resources are available for case management plans and greeting management; existing configurations are unaffected unless you adopt them.
- **Greetings:** Export/import now includes audio files via `audio_filename` and `audio_file_content_hash`; review exported assets after upgrade.
- **Integration actions:** Static built-in actions appear as data sources in exports; update references if you re-export, and use `integration_id` on the data source when action names collide.
- **Integration credentials:** Re-export may include credentials that were previously skipped due to naming format.
- **Outbound campaign rules:** New v2 condition groups and scheduling fields are optional; existing rules using `campaign_rule_conditions` continue to work. Remove `sub_type_settings` from call, chat, email, and callback queue media blocks if present — Terraform will plan to destroy those values.
- **Routing queues:** New alert-tone and per-subtype inactivity settings are optional; removing unsupported `sub_type_settings` from non-message media blocks may show plan diffs.
- **Auth roles with wildcard permissions:** Roles using wildcard `action_set` values should see fewer diffs on `permission_policies.conditions` after refresh.
- Configurations not using the changed resources are likely unaffected.
