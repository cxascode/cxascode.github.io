### Upgrade impact
- **New resources:** Five resources are newly registered and available for use: case management stageplans/stepplans/caseplans, group greetings, and user greetings.
- **Outbound campaign rules:** Existing rules using `campaign_rule_conditions` continue to work unchanged. To adopt v2 condition groups, set `campaign_rule_processing = "v2"` and migrate conditions into `condition_groups`. New scheduling condition types are available for time-based rules.
- **Outbound campaigns:** Preview campaigns can now set `agent_owned_column`; `max_calls_per_agent_decimal` is populated on read for visibility into API-returned decimal values.
- **Routing queues:** Configurations using `sub_type_settings` under `call_media_settings` or `callback_media_settings` must remove those blocks; subtype settings belong under `message_media_settings`. New alert-tone and per-subtype inactivity timeout settings are available where supported.
- **Integration actions:** Data source lookups for ambiguous action names can pass `integration_id`. Publishing published integration actions should behave correctly after the version fix.
- **Auth roles:** Roles with wildcard permission policies and `conditions` should see fewer spurious diffs after refresh.
- Configurations not using the changed resources are likely unaffected.
