### Upgrade impact
- Two new resources are available: `genesyscloud_conversations_messaging_integrations_whatsapp` for WhatsApp messaging integrations and `genesyscloud_task_management_worktype_status_transition` for worktype status transition rules.
- `genesyscloud_knowledge_document` no longer publishes documents through the `published` attribute; create variations with `genesyscloud_knowledge_document_variation` instead. Existing configurations that set `published = true` should be reviewed after upgrade.
- Status transition settings on `genesyscloud_task_management_worktype_status` are deprecated; migrate `destination_status_ids`, `default_destination_status_id`, `status_transition_delay_seconds`, and `status_transition_time` to `genesyscloud_task_management_worktype_status_transition`.
- `genesyscloud_routing_queue` `users` is no longer populated from the API when omitted; existing stacks that relied on computed queue members may need explicit `users` configuration or should omit the attribute intentionally.
- `genesyscloud_oauth_client` adds placeholder attributes and optional secret file output to simplify linking OAuth clients with `genesyscloud_integration_credential` resources.
- `genesyscloud_outbound_campaign` dynamic contact queueing settings gain a `filter` attribute and relaxed `sort` requirements; changing `sort` or `filter` replaces the campaign.
- `genesyscloud_recording_media_retention_policy` supports optional `team_ids` conditions across policy types.
- The `genesyscloud_task_management_worktype_status` data source ID format changed; update any references that assume the old ID shape.
- Bulk export may produce different Terraform block labels for flows, integrations, knowledge documents, users, and other resources when names collide or when `ENABLE_BCP_OPTIMIZED_SANITIZER` or `ENABLE_OPTIMIZED_SANITIZER` is set; bulk export for `genesyscloud_outbound_contact_list` contacts waits longer with exponential backoff before retrieving signed export URLs; bulk export for `genesyscloud_recording_media_retention_policy` resolves `team_ids` to `genesyscloud_team` references.
- Most other existing configurations are likely unaffected.
