### Upgrade impact
- Existing configurations are likely unaffected for most resources.
- `genesyscloud_business_rules_decision_table` is newly available for managing business rules decision tables.
- `genesyscloud_outbound_callabletimeset` no longer requires `callable_times`; remove it only if you intentionally want an empty callable time set.
- `genesyscloud_outbound_campaign` resources in `complete` status can be re-enabled by setting `campaign_status` to `on`; plans will now show that transition. Destroy operations rely on live API status rather than Terraform state.
- Destroying `genesyscloud_knowledge_knowledgebase` resources may take longer while dependency tracking rebuild runs.
- `genesyscloud_quality_forms_survey` configurations can use the newly supported `language` values where your organization supports them.
- DID pool phone numbers that were rejected for exceeding country length limits may now pass validation if they are valid, correctly formatted E.164 values.
