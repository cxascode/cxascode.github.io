### Resources changed

#### genesyscloud_webdeployments_configuration

Added:
- `messenger.file_upload.enable_attachments` — Whether file attachments are enabled in messenger.
- `messenger.session_persistence_type` — Messenger session persistence mode (`AcrossSubdomains` or `DomainOrSubdomainOnly`).
- `messenger.apps.conversations.notifications` — Notification settings for messenger conversations, including whether notifications are enabled and the notification content type (`IncludeMessagesContent` or `ExcludeMessagesContent`).
- `messenger.apps.conversations.session_duration_seconds` — Guest session duration in seconds for messenger conversations.
- `support_center.label_filter` — Optional document label filter for the knowledge portal; limits returned documents to those with at least one of the specified labels.
- `support_center.label_filter.label_ids` — Knowledge label IDs used by the support center label filter.

Removed:
- None detected.

- Export behavior — Reference resolution paths updated for `enabled_categories.category_id`, `apps.knowledge.knowledge_base_id`, and `support_center.label_filter.label_ids`.
- Export behavior — `support_center.knowledge_base_id` is no longer resolved to a `genesyscloud_knowledge_knowledgebase` reference during export.

### Upgrade impact
- Existing Terraform configurations are likely unaffected unless you use the newly added `genesyscloud_greeting` resource, web deployment resources, AI Studio summary settings, telephony edge groups or sites, routing queues with message media settings, or provider export.
- `genesyscloud_greeting` is newly available for managing audio greetings.
- Web deployment configuration and deployment resources gain messenger notification, file attachment, session, push integration, and knowledge portal label filter settings.
- `genesyscloud_ai_studio_summary_setting` now treats omitted `pii.all` as `true`; review configurations that relied on the previous implicit `false` default.
- `genesyscloud_telephony_providers_edges_site` updates should succeed in large orgs that previously hit HTTP 413 errors.
- `genesyscloud_routing_queue` plans may show fewer spurious diffs for `media_settings_message.sub_type_settings` ordering.
