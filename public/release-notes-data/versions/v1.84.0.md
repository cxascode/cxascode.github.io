### Provider configuration changes
- `max_concurrent_pages` — New optional setting (default `1`) to fetch list API pages in parallel during large reads; set `GENESYSCLOUD_MAX_CONCURRENT_PAGES` and increase `token_pool_size` alongside higher values.
- `token_pool_size` — Maximum allowed value increased from 20 to 50; description now documents OAuth startup cost, rate-limit risk, and guidance to align pool size with `max_concurrent_pages`.

### Upgrade impact
- Existing configurations are likely unaffected unless you use deprecated Predictive Engagement webchat actions, outbound digital rule sets with only one content template, callable-time or contactable-time column names on outbound contact lists, or Windows-style file paths on grammar or response asset resources.
- Optional: set `max_concurrent_pages` above `1` (and raise `token_pool_size` to match) to speed up paginated reads for flows, groups, queues, skills, scripts, and users.
- Review `genesyscloud_journey_action_map` resources using `media_type = "webchat"` or `action.is_pacing_enabled`; both are deprecated ahead of ACD Chat v2.0 removal.
- Re-export task-management and outbound digital ruleset resources if you rely on reference resolution for status IDs or content template IDs.
- Windows path normalization on `genesyscloud_architect_grammar_language` and `genesyscloud_responsemanagement_responseasset` should reduce spurious plan diffs after upgrade.
